# Chat 页面舒适度焕新设计规格

> 日期: 2026-06-02 | 状态: 设计中 | 目标文件: chat.html

---

## 1. 设计目标

以"舒适优先"重新设计 chat.html 前端页面。保持猫咪主题、玻璃态风格和所有现有功能，但从布局、配色、氛围三个维度注入"温暖小窝"的沉浸感。

**核心原则:**
- 聊天区居中固定宽度，不被任何面板挤压
- 降低色彩饱和度约 30%，追求"窝在沙发里翻书"的放松感
- 减少视觉噪音，头部/输入区极简化

---

## 2. 布局架构

### 2.1 整体结构

```
┌──────────────────────────────────────────────┐
│  顶部窄条 (全宽): 标题 + 模式切换胶囊 + 设置   │
├────┬──────────────────────────────┬──────────┤
│ 左 │                              │ 右       │
│ 留 │    聊天区 max-width: 680px    │ 留       │
│ 白 │    margin: 0 auto (居中)      │ 白       │
│ 56 │                              │ 56       │
│ px │  ┌──────────────────────┐    │ px       │
│    │  │  欢迎卡片 / 消息流    │    │          │
│    │  │  输入区 (同宽居中)    │    │          │
│    │  └──────────────────────┘    │          │
│ 标 │                              │ 标       │
│ 签 │                              │ 签       │
├────┴──────────────────────────────┴──────────┤
└──────────────────────────────────────────────┘
```

### 2.2 布局规则

| 规则 | 值 |
|------|-----|
| 聊天区最大宽度 | `max-width: 680px` |
| 聊天区水平定位 | `margin: 0 auto` (居中) |
| 两侧留白 | `flex: 0 0 56px` — 给标签预留空间 |
| 输入区定位 | 同聊天区宽度，居中 |
| 头部 | 全宽，内容居中对齐 |

### 2.3 抽屉/侧栏行为 (保留现版机制)

- **左侧抽屉 (历史对话):** `position: absolute`，宽度 240px，从左侧滑出。展开时覆盖在留白区之上，聊天区宽度不变。
- **右侧抽屉 (提示词):** `position: absolute`，宽度 260px，从右侧滑出。同上叠加逻辑。
- **边缘标签:** 保留现版竖排文字标签（"历史" / "提示词"），位于 56px 留白区内。点击展开/收起，与现版交互一致。
- **手机端 (<768px):** 留白消失，聊天区全宽。标签改为浮动圆形按钮。抽屉全屏覆盖 + backdrop。

### 2.4 与现版布局的关键差异

| 项目 | 现版 | 新版 |
|------|------|------|
| 侧栏默认状态 | 左侧栏默认打开 (240px) | 两侧均默认关闭 (聊天区完整居中) |
| 聊天区宽度 | flex:1 自动填充，被侧栏挤压 | 固定 max-width 680px 居中，永不挤压 |
| 头部元素 | 返回链接 + 猫耳朵 + 标题 + 副标题 + 模式切换 + 设置 | 设置图标 + 标题 (可点击回首頁) + 模式切换胶囊 |
| 输入区布局 | 全宽 | 同聊天区宽度居中 |

## 3. 配色方案 — "温暖小窝"

### 3.1 CSS 变量映射

```css
:root {
    /* 背景 — 温暖米纸色系 */
    --bg-color: #fef9f0;          /* 米纸白 (原 #fff5f0) */
    --box-bg: #ffffff;

    /* 主色调 — 降饱和暖杏橘 */
    --primary-color: #d49478;     /* 暖杏橘 (原 #ff8a65) */
    --primary-dark: #b86840;      /* 深杏 (原 #e65100) */
    --primary-light: #e8c8b0;     /* 浅杏 (原 #ffcc80) */
    --primary-bg: #fdf3e4;        /* 杏白底 (原 #fff3e0) */

    /* 文字 — 可可棕色系 */
    --text-color: #5d3a1a;        /* 可可棕 (不变) */
    --muted-text: #a08c7a;        /* 淡墨灰 (原 #a08c7a) */

    /* 面板 — 棉絮质感 */
    --soft-panel: rgba(255,250,245,0.55);   /* 暖调面板 */
    --soft-border: rgba(200,160,120,0.12);  /* 亚麻边框 */

    /* 代码模式保持不变 (蓝色系不受影响) */
}
```

### 3.2 气泡配色

| 气泡类型 | 背景 | 边框 | 文字 |
|---------|------|------|------|
| AI 消息 | `rgba(255,255,255,0.5)` + blur | `rgba(200,160,120,0.08)` | `#5d3a1a` |
| 用户消息 | `linear-gradient(135deg, #d49478, #c47a5a)` | 无 | `#fff` |
| AI 消息 (代码模式) | `rgba(255,255,255,0.5)` | `rgba(74,144,217,0.08)` | `#1a3a5c` |
| 用户消息 (代码模式) | `linear-gradient(135deg, #4a90d9, #357abd)` | 无 | `#fff` |

### 3.3 柔光背景球

- 尺寸增大约 30% (600px → 800px)
- 透明度从 0.45 降至 0.22–0.28
- 颜色从高饱和橙/粉转为暖杏/亚麻色
- 漂移动画速度放慢约 20%
- 保留噪点纹理层 (opacity: 0.025)

## 4. UI 区域设计

### 4.1 头部 (chat-header)

**精简内容:**
- 左侧: 无返回链接（用户使用浏览器后退即可）。如需保留入口，仅保留标题本身作为可点击的首页链接。
- 中间: "🐱 和小橘聊天" 标题
- 模式切换: 猫咪 🐱 / 认真 💡 胶囊切换器 (保留现版交互和极光滑动效果)
- 右侧: ⚙️ 设置齿轮按钮 (保留现版 click → rotate → 打开设置抽屉)

**去掉:**
- 猫耳朵 CSS 三角形装饰
- 副标题 "AI 猫娘小橘 — 站长的小猫咪"
- "🐾 回喵码" 返回链接 (如需保留，改为顶部左上角极小文字链接)

**高度:** 从 ~80px 缩减到 ~48px

### 4.2 聊天区 (chat-wrapper)

- `max-width: 680px; margin: 0 auto;`
- 消息间距 (`--msg-gap`) 同现版: 12px
- 欢迎卡片: 保留，风格更新为暖杏调
- 滚动行为不变

### 4.3 输入区 (input-area)

**精简内容:**
```
📎 [输入框................] [➤]
    🌐 搜索  🧠 思考
```

- 附件按钮 + 输入框 + 发送按钮: 一行水平排列
- 搜索/思考开关: 从按钮改为输入框上方的微型胶囊标签 (更内敛)
- 展开按钮 (↕): 保留，放在输入框右上角
- 文件预览: 保留在输入框上方
- 输入区背景: 半透明玻璃态 + 顶部细边框

**去掉:**
- 输入区不再全宽，改为同聊天区宽度居中

### 4.4 侧边抽屉

- 保留现版结构和功能: 头部、列表、新建按钮、关闭按钮、底部用户信息
- 材质更新为温暖小窝配色
- 宽度不变 (左 240px, 右 260px)
- 交互逻辑完全不变

### 4.5 设置面板 (settings-drawer)

- 保留完整功能和结构
- 配色跟随主题变量更新 (自动适配)

### 4.6 其他组件

| 组件 | 变更 |
|------|------|
| 打字指示器 | 背景和边框色更新，动画不变 |
| 深度思考卡片 | 边框色从橙改为暖杏调 |
| 搜索引用卡片 | 保持不变 (青色系) |
| 认证弹窗 | 配色更新 |
| 提示词编辑器 | 配色更新 |
| Toast | 不变 |
| Lightbox | 不变 |

## 5. 动效设计

### 5.1 保留
- 消息出现动画 (`msgPop`)
- 打字点动画 (`dotWave`)
- 模式切换极光滑块
- 设置齿轮旋转
- 侧栏滑入/滑出 (cubic-bezier 0.34, 1.56, 0.64, 1)
- 发送按钮 3D 按压效果
- 柔光球漂移动画

### 5.2 调整
- 柔光球漂移速度: 减慢约 20% (18s→22s, 22s→26s)
- 欢迎卡片猫咪弹跳: 从 `ease-in-out infinite alternate` 改为更柔和的 `ease-in-out infinite` (非交替)

## 6. 响应式设计

### 6.1 桌面 (>768px)
- 标准居中布局，两侧留白 56px

### 6.2 平板/手机 (≤768px)
- 两侧留白消失 (`flex: 0 0 0`)
- 聊天区 `max-width: 100%`，全宽
- 边缘标签改为浮动圆形按钮 (保留现版 mobile-sidebar-btn 样式)
- 抽屉全屏覆盖 + backdrop (保留现版逻辑)
- 头部进一步压缩

### 6.3 小屏手机 (≤480px)
- 消息气泡 `max-width: 95%`
- 字体略缩小
- 输入区按钮尺寸微调

---

## 7. 兼容性要求

- **代码模式 (body.code-mode):** 所有蓝色系配色保持不变，仅暖色系组件受影响
- **设置面板:** 字体大小/消息间距/气泡圆角等外观设置继续生效
- **认证:** Auth 逻辑完全不动
- **API:** 所有后端通信逻辑不动
- **浏览器:** 保持现有兼容性 (Chrome, Safari, Firefox, Edge)
- **动画:** GSAP 依赖不变，CSS animation 不变

## 8. 实现范围

### 修改文件
- **chat.html** — 主要改动 (~2497 行，预计改动 CSS 约 300 行 + HTML 结构调整)

### 不改动
- server.js
- index.html
- data/ 目录
- chat.json
- package.json

### 风险点
1. CSS 变量重命名可能影响代码模式下未覆盖到的硬编码颜色
2. 布局从三栏改居中单栏需要仔细处理 flex 结构
3. 边缘标签的 position 需要随着聊天区居中而重新计算
